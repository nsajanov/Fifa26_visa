# ⚽ WC 2026 Playoff Predictor — Telegram-бот (полностью автоматический)

Прогнозный турнир по **плей-офф** Чемпионата мира 2026 прямо в Telegram.

- **Групповой этап = только факты.** Игроки смотрят реальные результаты и таблицы групп (`/facts`).
- **Игра начинается с плей-офф.** Все прогнозируют одну и ту же реальную сетку Round of 32 → финал. Старт равный для всех, дальше копят очки.
- **Всё само.** Раз в день бот сам подтягивает реальные счета из football-data.org, пересчитывает очки и сам публикует лидерборд. Админу вручную обновлять ничего не нужно.

> Программировать не нужно — код готов. Ниже инструкция «по кнопкам».

---

## Очки (плей-офф)
- Угадал победителя матча: R32 +3 · 1/8 +5 · 1/4 +8 · 1/2 +12 · бронза +5 · **финал/чемпион +25**.
- Бонус за счёт (если угадал и саму пару команд матча): **5** точный · **3** разница · **1** исход.
- Настраивается в `scoring.py` (`KO_WIN_POINTS`).

## Файлы
`bot.py` — бот и ежедневный авто-отчёт · `results_api.py` — подтяг и разбор результатов football-data.org · `scoring.py` — подсчёт · `sheets.py` — хранение (Google Sheets/локально) · `fixtures.py` — сетка · `webapp/index.html` — мини-приложение прогноза.

## Запуск за 6 шагов

### 1) Бот
@BotFather → `/newbot` → возьми **токен**.

### 2) Бесплатный ключ для авто-результатов  ⭐ главное
Зайди на **football-data.org** → Register (бесплатно) → в личном кабинете будет **API token**. Это `FOOTBALL_DATA_TOKEN`. Именно он делает всё автоматическим.

### 3) Google Sheets (хранилище + наглядная таблица)
1. Создай пустую таблицу, скопируй **ID** из ссылки (между `/d/` и `/edit`).
2. console.cloud.google.com → новый проект → включи **Google Sheets API** → Credentials → **Service account** → Keys → Add key → **JSON**.
3. В JSON найди `client_email` и дай этому адресу доступ «Редактор» к таблице (кнопка «Поделиться»).
4. Весь JSON → переменная `GOOGLE_CREDENTIALS_JSON`, ID → `SPREADSHEET_ID`.
   *(Можно пропустить для теста — данные лягут в `local_db.json`.)*

### 4) Мини-приложение (GitHub Pages, бесплатно)
Загрузи `webapp/index.html` как `index.html` в новый репозиторий → Settings → Pages → Branch `main`. Получишь ссылку → это `WEBAPP_URL`. В @BotFather: `/setmenubutton` → пришли ту же ссылку.

### 5) Хостинг бота (Railway, бесплатно)
railway.app → New Project → Deploy from GitHub (загрузи эту папку). В **Variables** впиши всё из `.env.example`:
`BOT_TOKEN`, `WEBAPP_URL`, `ADMIN_IDS` (твой id — узнать у @userinfobot), `GROUP_CHAT_ID`, `FOOTBALL_DATA_TOKEN`, `REPORT_HOUR`, `SPREADSHEET_ID`, `GOOGLE_CREDENTIALS_JSON`. Railway сам запустит `python bot.py` (см. `Procfile`).

### 6) Старт
`/start` боту → `/setdeadline 2026-06-28 16:00` (закрытие приёма перед первым матчем плей-офф). Разошли ссылку на бота коллегам.

## Что происходит дальше (само)
- Каждый день в `REPORT_HOUR:00` бот тянет результаты, обновляет таблицы групп (факты) и сетку плей-офф, пересчитывает очки и постит лидерборд в общий чат.
- Игроки: `/facts` — результаты и группы · `/me` — мои очки · `/leaderboard`.

## Админ (обычно НЕ нужно — всё авто)
- `/sync` — подтянуть результаты прямо сейчас (проверить).
- `/win 73 Brazil` — вручную поправить победителя матча, если API задержал данные.
- `/post` — опубликовать таблицу немедленно.

---

## Честно про автоматику
football-data.org отдаёт результаты ЧМ; бот раскладывает плей-офф **без ручной разметки** (каждый матч R32 «привязывается» по известной позиции, дальше — по победителям). Это единственная внешняя зависимость. Если по новому формату Round of 32 в API будут расхождения в названии стадии/команд — поправь словарь `STAGE_CODES`/`ALIAS` в `results_api.py` (одна строка) или используй `/win` как страховку. Всё остальное — полностью автоматически.
